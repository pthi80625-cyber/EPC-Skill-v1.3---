# EPC 工程总承包项目招标研判 Skill —— 使用说明

## 简介

本 Skill 是一个针对 **中国境内 EPC（设计-采购-施工总承包）招标文件** 的专业分析工具。它模拟了拥有15年以上经验的总经济师/副总经理的研判逻辑，能够快速输出结构化风险评估报告，帮助施工企业、联合体牵头方在投标决策阶段识别关键风险、制定谈判策略。

## 核心特点

- **中国特色**：重点关注垫资、联合体强制捆绑、评定分离、农民工工资保证金、地方保护、异常低价等本土化风险点。
- **八步法覆盖全流程**：从资格筛查、资金垫资、工期合理性、合同漏洞、联合体策略、异常低价，到**业主画像与谈判话术**、**领导层一分钟汇报**，一应俱全。
- **可进化**：每次分析后会主动询问“遗漏或需调整的点”，并记录至进化日志，后续调用自动加载更新，持续贴合用户实际需求。

## 适用场景

- 施工单位、设计院、联合体牵头方的 **市场部/投标部/总经办** 在获取招标文件后的快速研判。
- 副总/总经济师在 **谈判前准备** 话术和底线清单。
- 向董事长/总经理 **紧急汇报** 时，获取一分钟电梯演讲内容。

## 使用方法

### 方式一：在支持 Codex 的环境（如本对话）中直接调用

1. 将本 Skill 的内容（`EPC_Bid_Analysis_Skill_v1.3.md`）完整告知 AI 模型。
2. 上传招标文件（PDF/Word/截图），或直接粘贴关键内容。
3. 发送指令：**“按 EPC 研判 skill 分析”**。
4. AI 将严格按照八步法输出结构化报告，最后会主动询问您的反馈以进化记忆。

### 方式二：内部人工对照使用

即使不通过 AI，您也可以将 Skill 中的 **八步法清单** 打印出来，人工逐项核对招标文件，避免遗漏重要风险。

## 进化机制说明

- **每次分析后**，AI 会主动提问：“本次分析是否有您特别关注而我遗漏的点？或您认为哪些部分需要调整权重？”
- **您反馈后**，AI 会在 Skill 的“进化日志”区域记录（如 v1.4 用户要求增加“评标办法中的分值陷阱分析”）。
- **下次调用**时，AI 会自动加载最新日志，使分析更贴合您的实际关注点。

> 建议您将进化日志视为团队的“投标经验知识库”，长期积累后形成公司专属的研判规则。

## 常见问题

**Q：这个 Skill 只适用于燃气管道项目吗？**  
A：不，它适用于所有 EPC 工程总承包项目，包括市政、房建、电力、水利、环保等。只需在具体分析时根据专业调整“类似工程业绩”的描述。

**Q：如果招标文件不是 EPC 模式（如仅施工标），能用吗？**  
A：可以，但部分条款（如设计责任、EPC 特有风险）可跳过或弱化。后续进化时可增加“模式识别”功能。

**Q：Skill 中的谈判话术直接照搬给业主是否合适？**  
A：建议根据您与业主的实际关系调整语气，但核心提问点和底线原则强烈建议保留。
